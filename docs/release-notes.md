# Release notes

Этот документ хранит пользовательские заметки к релизам Chaotika. Его нужно
обновлять для каждого production rollout и для каждой публикации store build.

Пишите заметки так, чтобы их можно было почти без правки перенести в App Store,
Google Play, RuStore или в сообщение пользователям.

## Как вести

- В `Unreleased` собираются изменения, которые еще не ушли в production.
- Перед релизом перенесите пункты из `Unreleased` в новый раздел
  `vX.Y.Z - YYYY-MM-DD`.
- Укажите каналы релиза: `Web`, `PWA`, `iOS`, `Android`.
- Описывайте пользовательский эффект, а не внутреннюю реализацию.
- Breaking changes, ручные действия пользователя и ограничения выносите отдельно.
- Если изменение не заметно пользователю, но важно для поддержки, поместите его
  в `Техническое`.

## v1.0.4 - 2026-05-13

Каналы: Web, PWA, iOS, Android

История собрана ретроспективно по коммитам `04b1956` - `51d47cb`, включая
приложение, API, PWA и native shell. Merge-коммиты и dependency-only bump'ы не
вынесены отдельными пользовательскими пунктами.

### Новое

- Запущена базовая версия Chaotika: экран «Сегодня», таймлайн, сферы жизни,
  создание и редактирование задач, планирование по датам и времени, дедлайны,
  важность, иконки и заметки.
- Добавлен дневной фокус: задачи на сегодня, просроченные задачи, задачи на
  завтра, выполненное за день и режимы нагрузки для более реалистичного плана.
- Добавлены сферы жизни с недельным балансом, статистикой активности,
  подсветкой забытых сфер и отдельными страницами сфер.
- Добавлен список покупок с активными и завершенными позициями, быстрым
  добавлением, удалением и синхронизацией.
- Добавлен планировщик уборки: зоны, шаблоны зон, расписание по дням недели,
  задачи уборки с глубиной, энергией, приоритетом, исполнителем, сезонностью,
  пропуском, переносом и завершением.
- Добавлены рутины и привычки: регулярные задачи, статистика выполнения,
  streak'и, показатели за месяц и быстрый доступ из мобильной навигации.
- Добавлены общие пространства: создание, переключение, переименование и
  удаление shared workspace, управление участниками, роли и ограниченный набор
  экранов для совместной работы.
- Добавлены профиль пользователя, аватар, управление данными аккаунта и
  отдельная страница профиля.
- Добавлена админка для пользователей, ролей, активности, количества задач,
  настроек и загрузки пользовательских иконок.
- Добавлена библиотека иконок и загрузка icon assets для задач, сфер и
  пользовательских сценариев.
- Добавлена PWA-поддержка: manifest, service worker, иконки и installable web
  experience.
- Добавлены нативные iOS и Android shell через Capacitor с мобильной
  навигацией, splash/icon assets и синхронизацией web bundle.
- Добавлены Android widgets для задач на сегодня и таймлайна: стилизованные
  карточки, действия, настройка внешнего вида, прозрачность фона и запуск
  создания задачи из виджета.
- Добавлены push-уведомления и backend-напоминания по задачам.
- Добавлена интеграция с Алисой: account linking, создание задач голосом,
  добавление покупок, чтение задач на сегодня/завтра, гибридный парсер команд и
  LLM fallback.
- Добавлена авторизация Chaotika: email/password, восстановление пароля,
  refresh-сессии, mobile session storage и OAuth-flow для Алисы.

### Улучшено

- Переработана мобильная навигация: основные разделы вынесены ближе к
  повседневным сценариям, остальные экраны доступны через «Еще».
- Привычки переведены в модель регулярных задач, чтобы рутины работали вместе с
  обычным планированием, статистикой и экраном «Сегодня».
- Улучшены карточки задач, форма создания задачи и поля рутины: добавлены более
  точные состояния, метаданные и редактирование без лишних переходов.
- Улучшены экраны сфер: баланс недели, статусы здоровья сфер и подсказки по
  областям без активности.
- Улучшена работа shared workspace: роли, доступы, участники и назначение задач
  стали согласованнее между API и web.
- Улучшен профиль и session bootstrap на web и mobile, включая восстановление
  сессии после перезапуска приложения.
- Улучшены Android widgets: визуальный стиль, кликабельные действия,
  настройки, обновление данных и отображение задач.
- Улучшены production deploy и mobile release workflow: появились проверки
  production API, синхронизация native bundle, version/build bump и сборка
  Android/iOS артефактов.

### Исправлено

- Исправлена гонка обновления mobile auth session, из-за которой приложение могло
  терять авторизацию при refresh.
- Исправлена production RLS role mismatch проблема, из-за которой
  authenticated API-запросы могли падать после deploy.
- Исправлены сценарии входа, восстановления сессии и повторной авторизации в
  web/mobile runtime.
- Исправлены ошибки shared workspace, ролей и отображения доступных экранов для
  shared runtime.
- Исправлены сценарии chaos inbox и списка покупок, включая обработку archived
  позиций и синхронизацию.
- Исправлены расчеты статистики сфер и отображение задач в карточках после
  переработки routine/habit логики.
- Исправлены mobile styles, safe-area и несколько проблем навигации после
  добавления PWA/native shell.
- Исправлены deploy-скрипты и проверки, чтобы релиз раньше ловил проблемы с
  production environment, миграциями, RLS и smoke-сценариями.

### Техническое

- Собран monorepo foundation: React/Vite web, Fastify API, shared contracts,
  TypeScript strict mode и workspace-пакеты.
- Добавлены PostgreSQL schema, миграции, seed, RLS-контекст, backup/migrate
  scripts и security checks.
- Добавлены OpenAPI generation/checks и shared API contracts для web/API
  границы.
- Добавлены offline store и sync-слои для planner, habits/routines и shopping
  list.
- Добавлены outbox, task reminders worker, push notification delivery и
  production smoke checks.
- Выделены shared API client на web и repository/route layers на backend, чтобы
  уменьшить дублирование и упростить поддержку.
- Добавлены CI, lint, format, typecheck, unit/integration/e2e проверки,
  coverage tooling и audit checks.
- Документированы architecture, shared workspace, release workflow, production
  deploy, store resubmission сценарии и incidents.

### Известные ограничения

- Production web/PWA deploy не обновляет уже установленные iOS/Android store
  builds: для native app нужен отдельный store release.
- История `v1.0.4` собрана задним числом по commit history, потому что в
  репозитории пока нет git tags для предыдущих релизных границ.
